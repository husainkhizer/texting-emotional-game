Four modes

Same level structure, different emotional framing and writing style.

Emotionally Gutting
Heavy, intimate, unresolved, reflective

Sarcastic Coworker
Petty, dry, passive-aggressive, office-specific

Absolute Humor
Chaotic, absurd, ridiculous, memeable

Balanced
Slightly serious, emotionally intelligent, broadest appeal

That is the right move because the player instantly understands:

same mechanic, different flavor

Best product framing

Call them Modes or Threads.

Example:

Gutting Mode

Coworker Mode

Chaos Mode

Balanced Mode

If you want slightly more premium naming:

Aftermath

Slack Thread

Unhinged

Between the Lines

That second naming system is stronger.

My recommendation for naming

Aftermath -- emotionally gutting

Slack Thread -- sarcastic coworker

Unhinged -- absolute humor

Between the Lines -- balanced, slightly serious

That feels branded.

How the 20 levels should work

You said you want 20 levels shared at once.

Best structure:

20 base scenarios

each scenario rewritten across all 4 modes

So instead of writing 80 unrelated levels, you write:

20 core moments

4 tonal versions of each

That gives you:

consistency

easier balancing

great replay value

Example core scenario

Base scenario:
“I didn’t think you would ___.”

Then by mode:

Aftermath

“I didn’t think you would stay.”

Slack Thread

“I didn’t think you would circle back.”

Unhinged

“I didn’t think you would survive that group chat.”

Between the Lines

“I didn’t think you would say that.”

Same structure. Different payoff.

That is how you make this scalable.

How each mode should feel
1. Aftermath -- emotionally gutting

Purpose:

emotional depth

reflection

strongest “Seen” ending

Tone:

quiet

intimate

painful but elegant

Examples:

“I should have told you before it turned into silence.”

“I kept waiting for the right moment, and then there wasn’t one.”

“I don’t know if you ever read what I meant.”

Reflection style:

philosophical

restrained

piercing

2. Slack Thread -- sarcastic coworker

Purpose:

relatable office comedy

passive-aggressive humor

highly shareable

Tone:

dry

sharp

realistic

mildly toxic in a funny way

Examples:

“Just circling back on the thing you said was ‘quick.’”

“I wasn’t sure if this needed a reply, but apparently nothing does.”

“Love the visibility. Unclear on the ownership.”

Reflection style:

cutting

funny

slightly too real

Example ending:

You communicated clearly, but unfortunately to coworkers.

3. Unhinged -- absolute humor

Purpose:

widest viral potential

absurdist comedy

chaos mode

Tone:

ridiculous

fast

overcommitted

meme-friendly

Examples:

“I didn’t mean to launch a blood feud in the family group chat.”

“You cannot just say ‘heard’ after what happened at Applebee’s.”

“I need you to know the raccoon was already inside.”

Reflection style:

chaotic but smart

weirdly insightful

super shareable

Example ending:

You pursued truth with the confidence of someone who had already lost everything.

4. Between the Lines -- balanced

Purpose:

broadest audience

best landing mode

strongest for first-time players

Tone:

emotionally grounded

thoughtful

slightly serious

accessible

Examples:

“I thought you understood what I was trying to say.”

“I wish I had been clearer earlier.”

“Some things changed before either of us named them.”

Reflection style:

reflective

measured

gently confronting

This should be the default mode on first launch.

Best UX structure
Home screen

Show all 4 modes as selectable cards.

Each card should communicate vibe fast:

Aftermath

For the words you should’ve said sooner.

Slack Thread

For office politics, dry replies, and calendar violence.

Unhinged

For chaos, nonsense, and terrible decisions.

Between the Lines

For thoughtful messages that hit a little too close.

That instantly sells the whole concept.

Best build approach

Do not build separate logic for each mode.

Build this data structure:

type Mode = "aftermath" | "slackThread" | "unhinged" | "betweenTheLines";

type LevelVariant = {
  prompt: string;
  options: { text: string; weights: Record<string, number> }[];
};

type SharedLevel = {
  id: number;
  scenario: string;
  variants: Record<Mode, LevelVariant>;
};

That way:

same level ID

same progression

different writing by mode

This is the cleanest architecture.

Best reflection structure by mode

Same engine, different voice.

Aftermath

Reflective and poetic

Slack Thread

Dry and observational

Unhinged

Chaotic and ridiculous

Between the Lines

Measured and insightful

So the system resolves:

outcome

pattern

tension

Then wraps it in the selected mode’s voice.

That is much better than building 4 different engines.

That gives you:

Very strong mix.
